IP版本4（IP v4）的 **TCP/IP (Transmission Control Protocol / Internet Protocol)** 是一个网络协议套件，Microsoft Windows使用它通过internet与其他计算机进行通信。它与Windows命名服务（如DNS）和安全技术（主要是IPsec）进行交互，因为这些服务有助于在机器之间成功和安全地传输IP数据包。

理想情况下，每当基于Windows的计算机通过网络进行通信时，都会使用TCP/IP。
本主题描述了TCP/IP协议套件的组件、协议体系结构、TCP/IP执行的功能、如何构造和分配地址以及如何构造和路由数据包。

Microsoft Windows Server 2003为传输控制协议/Internet协议（TCP/IP）套件提供了广泛的支持，它既是一种协议，也是一组用于IP互联网络连接和管理的服务。要正确理解基于IP的Windows Server 2003和Microsoft Windows 2000 intranets的配置、部署和故障排除，掌握TCP/IP的基本概念是绝对必要的。

## TCP/IP协议体系结构

TCP/IP协议映射到一个称为DARPA模型的四层概念模型，该模型以最初开发TCP/IP的美国政府机构命名。DARPA模型的四层是:应用层、传输层、互联网和网络接口。DARPA模型中的每一层都对应于七层开放系统互连（OSI）模型的一层或多层。

### 网络接口层

网络接口层（也称为网络接入层）处理在网络介质上放置TCP/IP数据包和从网络介质接收TCP/IP数据包。TCP/IP被设计为独立于网络访问方法、帧格式和介质。这样，TCP/IP可以用于连接不同类型的网络。其中包括局域网（LAN）介质（如以太网和令牌环网）以及广域网技术（如X.25和帧中继）。独立于任何特定的网络媒体允许TCP/IP适应新媒体，如异步传输模式（ATM）。

网络接口层包括OSI模型的数据链路层和物理层。请注意，Internet层没有利用网络接口层可能提供的排序和确认服务。假设网络接口层不可靠，通过会话建立、数据包排序和确认进行可靠通信是传输层的功能。

### 网络层/互联网层

互联网层处理寻址、打包和路由功能。互联网层的核心协议是IP、ARP、ICMP和IGMP。

- 互联网协议（IP）是一种可路由协议，用于处理IP寻址、路由以及数据包的分段和重组。
- 地址解析协议（ARP）将互联网层地址解析为网络接口层地址，例如硬件地址。
- 互联网控制消息协议（ICMP）负责提供诊断功能和报告因IP数据包传送失败而导致的错误。
- 互联网组管理协议（IGMP）处理IP多播组成员的管理。

互联网层类似于OSI模型的网络层。

### 传输层

传输层（也称为主机到主机传输层）负责为应用层提供会话和数据报通信服务。传输层的核心协议是传输控制协议（TCP）和用户数据报协议（UDP）。

- TCP提供一对一、面向连接的可靠通信服务。TCP处理TCP连接的建立、发送数据包的排序和确认以及传输过程中数据包丢失的恢复。
- UDP提供一对一或一对多、无连接、不可靠的通信服务。当要传输的数据量很小时（例如一个数据包中可以容纳的数据）、当您不想要建立TCP连接的开销时，或者当应用程序或上层协议提供可靠的传输时，使用UDP。

TCP/IP传输层包含OSI传输层的职责。
